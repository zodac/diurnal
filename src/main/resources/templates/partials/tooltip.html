{! Shared themed tooltip — the app's single tooltip style (`.app-tooltip` in app.css). Place INSIDE a host element that carries `group relative` (an icon button, or a wrapper around a control that can't hold children, e.g. an ). It reveals on hover; the Action-stats picker additionally toggles `.tip-open` for a mobile long-press. The host should ALSO carry an `aria-label` (this bubble is decorative — `aria-hidden`), so icon-only controls keep an accessible name. Params: text — the tooltip text (required) pos — 'top' (default) | 'bottom': which side of the host the bubble sits on align — 'center' (default) | 'left' | 'right': horizontal anchor. Use left/right for edge buttons so the bubble grows inward and can never push the page sideways. !} {! `pos`/`align` are optional (see above), so resolve them through the documented defaults here rather than requiring every caller to pass them — otherwise a caller that omits one makes the `{#if pos == ...}`/`{#if align == ...}` comparison below throw "Key not found" at render time (Qute is strict; a missing key is not silently falsy). `.or(default)` yields the default when the key is absent. NB: the elvis form (`pos ?: 'top'`) can't be used in a `{#let}` — its spaces break section-param parsing — so `.or(...)` is used here instead. !} {#let pos=pos.or('top') align=align.or('center')} {text} {/let}
