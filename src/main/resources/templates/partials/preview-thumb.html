{! A fixed-size, top-cropped screenshot thumbnail — the reusable UI element for ANY settings preview thumbnail. It guarantees a consistent size per viewport (see `.preview-thumb` in app.css for the ratios) by cropping whatever screenshot it is given to the TOP of a fixed frame, so the real aspect ratio of the source image is irrelevant. Drop this in wherever a settings thumbnail is needed; to resize every thumbnail, change `.preview-thumb` once. Every preview tile is a screenshot parameterised by (font, calendar style, theme); each picker FIXES one of those (its own dimension) and RENDERS all combinations of the other two, letting CSS reveal the one matching the user's live state — so a preview always mirrors the current config. Exactly one of the three mode params below is non-empty; the rest are "": • Theme mode (`themeBase` = "light"|"dark"|"system"): theme fixed. Renders the full-page shot per calendar style × font (`partials/preview-thumb-theme-group`). `#theme-options` reveals by selected style (`data-cal`) + active font (`.font-nova`). • Calendar mode (`calStyle` = "full"|"minimal"|"stacked"): style fixed. Renders the calendar-only shot per font × theme (`partials/preview-thumb-cal-group`). `#calendar-options` reveals by active font (`.font-nova`) + active mode (`.dark`). • Font mode (`fontBase` = "nova"|"standard"): font fixed. Renders the full-page shot per calendar style × theme (`partials/preview-thumb-font-group`). `#font-options` reveals by selected style (`data-cal`) + active mode (`.dark`). LOADING / SWITCHING. Two separate hiding mechanisms, on purpose: • VIEWPORT is gated with `display` (the `sm:` classes): the OTHER viewport's images are `display:none` and `loading="lazy"`, so they are never fetched — we don't download what the user can't see (a desktop user never sees the mobile thumbnails, or vice-versa, unless the window crosses `sm`). • The not-yet-shown variants WITHIN the current viewport (other fonts/styles/themes) are hidden with `visibility` (NOT display) by the `#…-options` rules in app.css, so they stay laid out and settings.html can `decode()` them up front. That makes font/style/theme switches flash-free; a `display:none` image cannot be pre-decoded (its decode() promise never resolves).
